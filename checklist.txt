    AC: with a scripted GrabFn, frame timestamps track start + k*interval within
        the tolerance asserted by tests/test_recorder_pacing.py; no burst-grab
        after a slow tick. (Regression of fix 722c127 — keep the test green.)
[x] ⚠ Capture path applies an exclusion hook before diff/storage  (pulled fwd from P3.2)
    AC: ScreenCapturer.blank_regions zeroes excluded areas on every grab, so an
        excluded region never produces a stored delta.
[x] Perceptual diff: bounded changed-region boxes (threshold=12, tile=48)
    AC: two disjoint changes -> exactly two boxes, each clamped to frame bounds;
        identical frames -> []. "Bounded" = one box per 8-connected changed-tile
        cluster, clamped (core/diff.py:changed_regions).
[x] Finalized .stut recording on clean stop   → capture loop, keystone
    AC: clean stop yields a single file at final_path; temp container gone;
        crash-before-stop leaves NO .stut (recorder.py BaseException unlinks temp).
[x] Size-comparison tool — baseline = uncompressed W*H*3 per tick (NOT a codec)
    AC: samples/generate_sample.py reproduces SIZE_COMPARISON.md byte counts
        exactly; the doc states the baseline is uncompressed raw, not H.264.
[x] Record measured size win   → size-comparison tool, finalized .stut
    AC: ratio <= stated target. If the target is missed (e.g. high-motion or a

Phase 3: hardening and privacy — safe load, no leaks on export
Container validation (was one checkbox; split into four testable guards):
[x]   (a) reject non-SQLite / malformed
      AC: truncated/garbage file -> "not a SQLite recording container".
[x]   (b) reject oversized blob / region / dimension
      AC: blob > MAX_BLOB_BYTES, region > MAX_REGION_PIXELS, or dim > MAX_DIMENSION
          each raises (assert each cap fires).
[x]   (c) reject decompression bomb BEFORE materialization
      AC: decode_region checks declared header dims vs MAX_REGION_PIXELS before
          convert("RGB")/np.array — verified in core/codec.py. Assert with a blob
          declaring absurd dims.
[—]   (d) path traversal — N/A for schema v1: the frames/meta tables carry no
          path/filename field, so there is nothing to traverse.  🛑 confirm before
          marking, and 🛑 if any future schema adds a path-bearing field.
[x] Region-exclude so chosen screen areas are never captured   → exclusion hook (P1)
    AC: with an exclusion region set, no stored box intersects it and reconstructed
        pixels there are blank (property test over a synthetic capture).
[x] Per-frame prev_hash chain verified on open   → keystone write-side
    AC: verify_chain() passes on the sample; flipping one stored blob byte makes it
        raise, naming the first broken frame.
Export with redaction (split from one checkbox):
[x]   Redaction rewrite (headless, tested)
      AC: export_recording with a redaction box zeroes those pixels in the output,
          gives the copy a fresh valid chain (verify_chain passes), and os.replaces
          atomically. Coverage is SPATIAL-ONLY — no content detection.
[x]   Export dialog with explicit confirm
      AC: UI refuses to write without an explicit confirm.  🛑 every export — human
          confirms the boxes cover the secret; 🛑 if dst path already exists
          (os.replace overwrites silently).
